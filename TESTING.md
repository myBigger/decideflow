# Testing Guide

## Quick Start

```bash
# 1. Install testing dependencies
npm install

# 2. Run all tests
npm test

# 3. Run tests in watch mode (auto-reload on changes)
npm run test:watch

# 4. Run with coverage report
npm run test:coverage
```

## Test Structure

```
src/
├── __tests__/
│   ├── setup.ts       # Vitest global setup (mocks Next.js, Supabase SSR)
│   ├── supabase.ts    # Mock Supabase client + seed data helpers
│   └── utils.tsx      # Test render helpers (renderWithProviders)
│
├── lib/
│   └── vote-engine.test.ts   # Pure function unit tests
│
└── app/api/decisions/
    └── api.test.ts            # API route integration tests
```

## Writing Tests

### Unit Tests (Pure Functions)

For files like `vote-engine.ts` — import and call directly:

```typescript
import { calculateVotingResults } from './vote-engine'
import { describe, it, expect } from 'vitest'

describe('my feature', () => {
  it('does the thing', () => {
    const result = calculateVotingResults(options, votes, members, 'simple')
    expect(result.passed).toBe(true)
  })
})
```

### API Route Tests

Use the `buildMockClient` helper from `supabase.ts` to populate an
in-memory store, then call the route handler directly:

```typescript
import { buildMockClient } from '@/__tests__/supabase'

const store = { decisions: [], options: [], votes: [], ... }
const mockClient = buildMockClient(store)

// Seed data
store.decisions.push({ id: 'd1', title: 'Test', ... })

// Override the factory
vi.mock('@/lib/supabase/server', () => ({
  createClient: () => mockClient,
}))

// Call route
const { GET } = await import('./route')
const res = await GET(new Request('/api/decisions'))
const body = await res.json()
expect(body.success).toBe(true)
```

### Component Tests

Use `renderWithProviders` to render with a mocked `AuthContext`:

```typescript
import { renderWithProviders, screen } from '@/__tests__/utils'

it('shows the title', () => {
  renderWithProviders({
    children: <MyComponent title="Hello" />,
    user: { id: 'u1', email: 'alice@example.com' },
  })
  expect(screen.getByText('Hello')).toBeInTheDocument()
})
```

## Mock Utilities (`@/__tests__/supabase.ts`)

| Function | Purpose |
|---|---|
| `makeDecision(overrides)` | Create a Decision fixture |
| `makeOption(content, overrides)` | Create an Option fixture |
| `makeVote(userId, optionId, overrides)` | Create a Vote fixture |
| `makeMember(userId, overrides)` | Create a TeamMember fixture |
| `buildMockClient(store)` | Full mock Supabase client with chain API |
| `createMockSupabase(store)` | Alternative typed mock client |

## Running Specific Tests

```bash
# Only vote-engine tests
npx vitest run src/lib/vote-engine.test.ts

# Only API tests
npx vitest run src/app/api/decisions/api.test.ts

# Watch mode for a specific file
npx vitest src/lib/vote-engine.test.ts
```

## Coverage

Coverage reports are generated by `npm run test:coverage`.
HTML report available at `coverage/index.html`.

Currently covered:
- `src/lib/vote-engine.ts` — 100% (core voting logic)
- API routes — auth checks, validation, business logic

Not covered (require full Next.js server):
- `src/app/**` page components (use Playwright/Cypress for e2e)
